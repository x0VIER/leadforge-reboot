# LeadForge Control Center: Navigation & Operation Guide

Welcome to your recovered LeadForge client acquisition command center. This folder contains all the files, trackers, and memory banks that sync your 500-leads database with the Elevor Flow website.

---

## 📂 Workspace Map

* **`LeadForge.code-workspace`**
  Double-click this file to open VS Code. It is configured to automatically mount both of your active directories side-by-side:
  1. **LeadForge CRM & Dossiers** (`C:\Users\loc9o\Desktop\AGR 1226`)
  2. **Elevor Flow Web App & Agents** (`C:\Users\loc9o\Desktop\Design, design, design\ai-ops-studio-site`)

* **`Recovered_Leads_Database.csv`**
  Your master CRM database of exactly 500 clean home service leads with phone numbers, emails, website dork links, and pre-audited digital conversion gaps.

* **`Active-Tasks-Tracker.md`**
  Your live task list. Keep track of what sprints have completed and what setup configurations (like wrangler authentication) are outstanding.

* **`Agent-Status-Board.md`**
  Real-time tracking of agent states (Hermes, Voss, Hale, Lang, Vale, Moreau).

* **`Trigger-Log.md`**
  The audit log of trigger events and database updates.

* **`Agents/` (Memory Folder)**
  Contains JSON definition files for each of your five active agents and the supervisor Hermes, detailing:
  - Personality profile
  - Base system prompt instructions
  - Complete memory of what tasks they completed on May 23 and May 25, 2026.

## 🔄 How to Resume & Trigger Work

1. **Open Workspace:** Double-click `LeadForge.code-workspace`.
2. **Authenticate Cloudflare:** Open the terminal in VS Code and run `npx wrangler login` to ensure deployment commands are authorized.
3. **Trigger Sourcing Lane:** Place a trigger payload (e.g. `sprint.trigger.json`) in the website's `agent_shared/triggers/` folder. This tells Hermes to start a new lead acquisition cycle.
4. **Outbox Review:** Outbound templates and qualified leads generated by your agents will automatically output to `agent_shared/outbox/` for your manual approval.
